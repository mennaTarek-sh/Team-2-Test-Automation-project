# Manual Testing Project

**This repository contains manual test cases designed for different user stories.**

All test cases are documented in Excel format and follow standard testing techniques.

__________________________________________________________________________________________

## 1. Credit Card Discounts

### User Story
As a customer, I want to open a credit card account so that I can receive discounts on my purchases.

### Acceptance Criteria
* New customers receive 15% discount on all purchases today.
* Existing customers with a loyalty card receive 10% discount.
* Customers with a coupon receive 20% discount (not combinable with new customer discount).
* Discounts are cumulative if applicable (e.g., Loyalty + Coupon = 30%).

### Deliverables
* Decision Table (8 scenarios compressed into 5 rules).
* Test Cases (9 detailed cases mapped to rules).

## 2. Add to Cart

### User Story
As a customer, I want to add a product to my cart so that I can purchase it.

### Acceptance Criteria
* Product must have at least one item in stock.
* If product is out of stock, user cannot add it.
* User must be logged in to add to cart.
* On adding, a message should confirm the item is added.
* The confirmation message should contain a valid link redirecting to the cart.

### Deliverables
* Traceability Table mapping acceptance criteria to rules.
* Test Cases including both valid and invalid scenarios.

## 3. Contact Us

### User Story
As a customer, I want to contact the website about a problem with my product.

### Acceptance Criteria
* Email field is mandatory and must be valid (`name@domain.xyz`).
* Contact name is mandatory.
* Message field is mandatory and must not exceed 500 characters.

### Deliverables
* Test Cases covering input validation (positive and negative cases).
